## Strict Completion Rule

Compile/import checks are not completion gates. They only prove the development
environment is coherent enough to continue. A milestone can be marked `DONE`
only when a real evidence manifest proves the design behavior against authentic
inputs, preferably PDFs from `/Users/luzi/aily_chaos/pdf`.

Required completion evidence path:

```text
~/.aily/runs/<run_id>/manifest.json
```

The manifest must identify source PDF paths and hashes, runtime commands, real
or mocked components, graph/vault/workflow artifacts, and milestone-specific
pass/fail reconciliation.

The manifest is necessary but not sufficient. Each gate also needs an evidence
matrix, an Obsidian vault review, and a cross-source reconciliation showing that
vault content, runtime results, durable records, events/logs, and direct
observations agree.

The lead agent must not manually create or modify evidence artifacts used to
pass a gate. Evidence files must include origin headers and must be generated by
the app, evidence runner, test harness, external service, database query/export,
or independent reviewer process.

Visible gate runbook:

```text
docs/AILY_TEST_QUALITY_GATES.md
```
